# UT0207 - Manual content edition

**Tactic:** [UTA02 - Technical Safeguarding](./README.md)

## Description

The handler may need to edit previously sent contents, either textual or multimedia. This may be used as a way of hiding contents sent by mistake or removing artifacts that exposed information which was not expected to be sent or that showed information wanted to be edited. Many messaging applications ensure that this information is edited at both ends of the conversation. However, depending on the sophistication of the source, it has to be assessed whether it is possible that he/she has technical knowledge to track edited content. Note that editing content sent by mistake may expose the information even if the source has not seen it after the edition. A typical scenario when this can happen is when using mobile messaging apps such as WhatsApp and Telegram. These applications can be configured to show a notification in the lock screen which is shown at the moment of receiving the message in the application and the message may not be updated when the edition takes place, revealing the original message only in the lock screen. Note that some applications like Signal may still show edit history as well as the edited disclaimer showing previous versions of the message.
