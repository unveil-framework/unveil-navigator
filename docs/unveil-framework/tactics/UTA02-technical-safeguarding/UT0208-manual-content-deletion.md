# UT0208 - Manual content deletion

**Tactic:** [UTA02 - Technical Safeguarding](./README.md)

## Description

The handler may need to delete previously sent contents, either textual or multimedia. This may be used as a way of hiding contents sent by mistake or removing artifacts that exposed information which was not expected to be sent or that showed information that should be deleted. Many messaging applications ensure that this information is edited at both ends of the conversation. However, depending on the sophistication of the source, they have to be assessed whether it is possible that they have technical knowledge to track edited content. Note that editing content sent by mistake may expose the information even if the source has not seen it after the edition. A typical scenario when this can happen is when using mobile messaging apps such as WhatsApp and Telegram. These applications can be configured to show a notification in the lock screen which is shown at the moment of receiving the message in the application and the message may not be updated when the edition takes place, revealing the original message only in the lock screen. Note that messages displayed as notifications may still be recoverable in a forensic investigation of the device.
